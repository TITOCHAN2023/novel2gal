# 资产生成方案

## 核心思路：分层生成

将视觉资产拆为**基础层**和**叠加层**，避免组合爆炸。

---

## 角色资产

### 基础层（预生成，高质量参考图）
- **角色基础体型**：全身六视图 + 脸部五视图（作为一致性锚点）
- **每套服装**：全身六视图 + 脸部五视图
- 资产量 = N 套服装 × 11 张参考图

### 叠加层（运行时合成）
- 受伤、脏污、血迹等动态状态
- 通过效果描述 + 局部重绘（inpainting）实现
- 不与服装做笛卡尔积

### 特殊处理
- 高频/标志性状态（如主角固定伤疤）可预生成参考图
- 临时性状态走叠加层即可

---

## 场景资产

### 基础层（预生成）
- 每个场景一张基准场景图（确定风格、光影、布局）

### 变体层（按需生成）
- 基于基准图生成不同视角、不同角色站位的变体
- 时间变化（白天/夜晚）、天气变化等

### 已知问题
- 场景变体的一致性问题（视角切换后细节可能不一致）
- 暂不阻塞产品设计，后续技术评审解决
- 即使有一致性偏差，体验仍然可接受

---

## 资产池：按需生成 + 复用机制

### 核心原则
1. **不预生成** — 需要时才生成
2. **生成后必持久化** — 带完整标注信息存入资产池
3. **优先查池复用** — 查到直接用，查不到再生成
4. 资产池随游玩逐渐丰富，命中率越来越高

### 角色立绘索引维度
`角色 × 服装 × 情绪 × 程度(1-10)`

示例：
- `角色A.校服.生气.3` → 已生成，直接复用
- `角色A.校服.生气.8` → 未生成，需要时生成并入池
- `角色A.战斗服.生气.3` → 未生成

程度 1-10 分级：避免只有一张表情 or 无限生成的两个极端。

### 场景索引维度
`场景 × 时间段(每4小时) × 世界状态`

时间段分 6 档：
- 凌晨(2-6) / 早晨(6-10) / 中午(10-14) / 下午(14-18) / 傍晚(18-22) / 深夜(22-2)

世界状态叠加：如 `公园.下午.花不存在` vs `公园.下午.花存在`

## 资产存储结构（对应图数据库节点）

```
character_角色名/
  base/              # 基础体型参考图（六视图+脸部五视图）
  outfit_校服/
    ref/             # 服装参考图（六视图+脸部五视图）
    emotion/         # 情绪立绘池（按需生成）
      生气_3.png
      微笑_5.png
  outfit_战斗服/
    ref/
    emotion/
  voice/             # 配音样本

location_场景名/
  base/              # 基准场景图
  time_早晨/          # 按时间段的变体（按需生成）
  time_傍晚/
  variants/          # 视角/站位变体（按需生成）
  ambient/           # 氛围音效
```
